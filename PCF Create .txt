Create PCF Request – Basic Information (Overview)

User can create a new PCF request by filling Basic Information first.

Page clearly shows Create PCF Request with PCF Request → New Request as reference text.

User can go back to the PCF list using < PCF Request.

Only one Save button is available at the bottom.

The process follows a step-by-step flow; users cannot skip steps.

Basic Information shows as Active while filling and Completed after saving.

Completed steps can be reopened and edited if needed.

Mandatory fields must be filled before moving forward.

Due Date is selected using a calendar; past dates are not allowed.

Clear error messages are shown if required information is missing.

Create PCF Request – Product Details (Functional Overview)

After saving Basic Information, the system automatically moves the user to Product Details.

The progress bar clearly shows which step is Completed, Active, or Pending.

Users must select Product Category, Component Category, and Component Type before proceeding.

Users can add or remove Product Specifications as needed.

Import BOM opens a new screen where users upload the BOM file.

BOM can be uploaded using Excel or CSV files only.

If a BOM already exists, the system asks for confirmation before replacing it.

Uploaded BOM data automatically fills Overview, Emissions, Transport, and Allocation sections.

BOM data is read-only; changes require editing and re-uploading the BOM file.

Users can freely switch between Overview, Emissions, Transport, and Allocation tabs.

Saving each section moves the user to the next section automatically.

After completing Allocation, Product Details is marked Completed and the user moves to Documentation.

All entered and imported data is saved and retained during navigation.
